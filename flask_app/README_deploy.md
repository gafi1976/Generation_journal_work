# Генератор журнала Lotus Notes — Установка на Windows Server

## Что нужно

- Windows Server 2019 / 2022 / 2025
- Python 3.10 или выше
- Доступ в интернет (только для установки, потом не нужен)

---

## Структура папки

```
journal_app/
├── app.py                  ← Flask приложение (основной файл)
├── run.py                  ← Запуск через Waitress
├── run.bat                 ← Запуск одним кликом
├── install.bat             ← Установка зависимостей
├── requirements.txt        ← flask, waitress
├── templates/
│   └── index.html          ← Веб-страница
├── _records_bank.py        ← Русский банк шаблонов
└── _records_bank_uz.py     ← Узбекский банк шаблонов
```

---

## Установка (один раз)

### Шаг 1 — Установить Python

1. Скачать с [python.org](https://www.python.org/downloads/) версию 3.10+
2. При установке **обязательно** поставить галочку **"Add Python to PATH"**
3. Проверить в командной строке:
   ```
   python --version
   ```

### Шаг 2 — Скопировать файлы на сервер

Скопировать всю папку `journal_app` на сервер, например:
```
C:\journal_app\
```

### Шаг 3 — Установить зависимости

Дважды щёлкнуть на файл:
```
install.bat
```
Или в командной строке:
```cmd
cd C:\journal_app
pip install -r requirements.txt
```

## Запуск

### Вариант 1 — Простой запуск (дважды щёлкнуть)

```
run.bat
```

### Вариант 2 — Командная строка

```cmd
cd C:\journal_app
python run.py
```

После запуска в консоли появится:

```
====================================================
  Генератор журнала Lotus Notes
  Сервер запущен: http://0.0.0.0:5000
  В браузере коллег: http://192.168.1.10:5000
====================================================
```

### Вариант 3 — Автозапуск при старте Windows (рекомендуется)

Открыть **Планировщик задач (Task Scheduler)**:

1. Создать задачу → "Создать задачу..."
2. **Общие**: название `JournalApp`, запускать для всех пользователей
3. **Триггеры**: при запуске системы
4. **Действия**: запустить программу
   - Программа: `C:\journal_app\run.bat`
   - Начать в: `C:\journal_app`
5. **Условия**: убрать галочку "Только при питании от сети"
6. Нажать OK

## Открыть в браузере

На сервере:
```
http://localhost:5000
```

На любом компьютере офиса (заменить IP на адрес сервера):
```
http://192.168.1.10:5000
```

Узнать IP сервера:
```cmd
ipconfig
```
Смотреть строку **IPv4-адрес**.

---

## Открыть порт в брандмауэре (если коллеги не могут зайти)

В командной строке от имени администратора:

```cmd
netsh advfirewall firewall add rule name="JournalApp" ^
  protocol=TCP dir=in localport=5000 action=allow
```

Или через GUI:
1. Пуск → **Брандмауэр Windows в режиме повышенной безопасности**
2. Правила для входящих подключений → Создать правило
3. Тип: **Порт** → TCP → Конкретный: **5000**
4. Действие: **Разрешить подключение**
5. Назвать: `JournalApp`

---

## Смена порта

Открыть `run.py` и изменить строку:

```python
PORT = 5000   # ← поменяйте на нужный, например 8080
```

---

## Частые вопросы

**Сервер запустился, но коллеги не могут зайти?**
→ Проверьте брандмауэр (см. выше). Убедитесь что используете правильный IP сервера.

**Как узнать IP сервера?**
→ Откройте командную строку на сервере и введите `ipconfig`

**Порт 5000 уже занят?**
→ Смените порт в `run.py` (например на 8080)

**Как остановить сервер?**
→ В консоли нажмите `Ctrl+C`

**Хочу чтобы сервер работал как служба Windows?**
→ Установите NSSM: https://nssm.cc/download
   ```cmd
   nssm install JournalApp "C:\journal_app\run.bat"
   nssm start JournalApp
   ```
